### [18:49] — Missing Package Errors After No Code Changes

**User reported:** App worked yesterday, no updates made, but today all pages show `ModuleNotFoundError` for `langchain_google_genai` and `ImportError` for `google.cloud.texttospeech`.

**Diagnosis:**
- User claimed to be running from activated venv, but `$VIRTUAL_ENV` was empty
- `run_app.sh` was missing venv activation — it called `streamlit` from global pyenv Python
- Venv packages verified present: `langchain-google-genai==2.1.2`, `google-cloud-speech==2.36.0`, `google-cloud-texttospeech==2.34.0`
- Global Python was missing these packages (only had `langchain-anthropic`, `langchain-openai`, etc.)

**Root cause:** Two-part issue:
1. `run_app.sh` never activated the venv before launching Streamlit
2. Even with venv activation, pyenv shims intercept the `streamlit` command and use global Python
3. The venv was incomplete — missing `streamlit` and many other packages from `requirements.txt`

**Fix applied:**
1. Installed all packages into venv: `.venv/bin/python -m pip install -r requirements.txt`
2. Updated `run_app.sh` to use `.venv/bin/python -m streamlit run app/main.py` instead of `streamlit run app/main.py`
   - This bypasses pyenv shims entirely and uses the venv's Python directly

**Lesson learned:** 
- When using pyenv, `source .venv/bin/activate` doesn't guarantee the venv's binaries are used — pyenv shims take precedence
- Always use `.venv/bin/python -m <module>` pattern for launcher scripts in pyenv environments
- The venv was never properly initialized with `pip install -r requirements.txt` — it only had manually installed packages

### [19:52] — New Feature: YouTube Thumbnail Generation

**User request:** Add AI-powered YouTube thumbnail generation to the Metadata page.

**Requirements:**
- Use metadata (titles + description) to create thumbnail-specific image prompt
- Generate catchy one-liner text overlay (3-7 words, ALL CAPS)
- Generate 16:9 thumbnail image via Vertex AI Imagen
- Display thumbnail preview in UI
- Provide download button for PNG file
- Reuse existing image generation infrastructure

**Implementation:**

**New module: `src/thumbnail_generation.py`**
- `generate_thumbnail_text(metadata)` → Uses Gemini LLM to create punchy 3-7 word text overlay
  - Temperature: 0.9 for creative output
  - Prompt engineered for click-worthy power words (REVEALED, SECRET, ULTIMATE, etc.)
- `generate_thumbnail_prompt(metadata, thumbnail_text)` → Creates detailed image prompt
  - Optimized for 16:9 composition with negative space for text overlay
  - Emphasizes high contrast, bold colors, photorealistic/cinematic style
- `create_thumbnail(metadata_path, output_path)` → Orchestrates full pipeline
  - Generates text → Generates prompt → Calls Vertex AI Imagen → Saves PNG + metadata JSON

**UI integration: `app/pages/4_metadata.py`**
- New section after hashtags: "🎨 YouTube Thumbnail"
- **Before generation:** "Generate Thumbnail" button
- **After generation:**
  - Full-width thumbnail preview
  - Shows generated text overlay
  - Expandable image prompt viewer
  - Download button (saves as `{project_name}_thumbnail.png`)
  - Regenerate button
- Real-time logging via `capture_stdout_to_streamlit`
- Lazy import of thumbnail module (only loads when button clicked)

**File outputs:**
- `projects/{name}/4_thumbnail.png` — Generated thumbnail image
- `projects/{name}/4_thumbnail_metadata.json` — Contains thumbnail_text, image_prompt, thumbnail_path

**Design decisions:**
- Thumbnail generation is **optional** — not part of approval flow
- Uses same Imagen model as video images (configurable via `config.json`)
- **Text is embedded directly by Imagen** — The image prompt explicitly instructs the model to render the text on the image with specified placement, style, and contrast
- Higher LLM temperature (0.9) for thumbnail text to encourage creative, punchy output
- Separate from video image generation — different prompt strategy optimized for thumbnails with text rendering

**Correction [20:00]:**
- Initial implementation had text generation separate from image rendering
- Updated to instruct Imagen to embed text directly on the image via the prompt
- Prompt now includes explicit text rendering instructions: placement, style, color, outline
