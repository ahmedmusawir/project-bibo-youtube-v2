### [18:49] — Missing Package Errors After No Code Changes

**User reported:** App worked yesterday, no updates made, but today all pages show `ModuleNotFoundError` for `langchain_google_genai` and `ImportError` for `google.cloud.texttospeech`.

**Diagnosis:**
- User claimed to be running from activated venv, but `$VIRTUAL_ENV` was empty
- `run_app.sh` was missing venv activation — it called `streamlit` from global pyenv Python
- Venv packages verified present: `langchain-google-genai==2.1.2`, `google-cloud-speech==2.36.0`, `google-cloud-texttospeech==2.34.0`
- Global Python was missing these packages (only had `langchain-anthropic`, `langchain-openai`, etc.)

**Root cause:** Two-part issue:
1. `run_app.sh` never activated the venv before launching Streamlit
2. Even with venv activation, pyenv shims intercept the `streamlit` command and use global Python
3. The venv was incomplete — missing `streamlit` and many other packages from `requirements.txt`

**Fix applied:**
1. Installed all packages into venv: `.venv/bin/python -m pip install -r requirements.txt`
2. Updated `run_app.sh` to use `.venv/bin/python -m streamlit run app/main.py` instead of `streamlit run app/main.py`
   - This bypasses pyenv shims entirely and uses the venv's Python directly

**Lesson learned:** 
- When using pyenv, `source .venv/bin/activate` doesn't guarantee the venv's binaries are used — pyenv shims take precedence
- Always use `.venv/bin/python -m <module>` pattern for launcher scripts in pyenv environments
- The venv was never properly initialized with `pip install -r requirements.txt` — it only had manually installed packages
