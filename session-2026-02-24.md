### [18:49] — Missing Package Errors After No Code Changes

**User reported:** App worked yesterday, no updates made, but today all pages show `ModuleNotFoundError` for `langchain_google_genai` and `ImportError` for `google.cloud.texttospeech`.

**Diagnosis:**
- User claimed to be running from activated venv, but `$VIRTUAL_ENV` was empty
- `run_app.sh` was missing venv activation — it called `streamlit` from global pyenv Python
- Venv packages verified present: `langchain-google-genai==2.1.2`, `google-cloud-speech==2.36.0`, `google-cloud-texttospeech==2.34.0`
- Global Python was missing these packages (only had `langchain-anthropic`, `langchain-openai`, etc.)

**Root cause:** `run_app.sh` never activated the venv before launching Streamlit.

**Fix applied to `run_app.sh`:**
- Added `source .venv/bin/activate` before `streamlit run app/main.py`

**Lesson learned:** Always verify venv activation in launcher scripts. The script worked before because user may have manually activated venv in terminal before running the script, but that's not reliable.
