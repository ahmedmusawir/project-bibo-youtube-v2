### Question 2: Stitch Designs for Streamlit?

**Partial yes.** Here's the reality:

| What Transfers | What Doesn't |
|----------------|--------------|
| Layout structure (sidebar, tabs) | Pixel-perfect styling |
| Component placement | Custom widgets |
| User flow / navigation | Animations, transitions |
| Color palette (roughly) | Complex interactions |

**My take:** Use Stitch designs as **wireframe reference**, not pixel targets. Streamlit has its own component library — fighting it wastes time.

Your `UI_SPEC.md` already translated the Stitch designs into Streamlit-native patterns. **That's your blueprint.**

#### **Architecture (Critical)**

```
project-bibo-youtube-v2/
├── src/                    # ← ENGINE (unchanged)

│   ├── text_to_speech.py

│   │   ├── project_selector.py
│   │   └── approval_buttons.py
│   └── tabs/               # One file per tab
│       ├── projects.py
│       ├── starter.py
│       ├── script.py
│       ├── audio.py
│       ├── images.py
│       └── video.py

```

**Key principle:** `app/` calls `src/` functions. UI and engine stay separate.
